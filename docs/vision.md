# Vision
## Overview
The Broad Learning Universal Education (B.L.U.E.) System is a system originally conceived and described by Sebastian Rey in his YouTube video [The "How to Do Anything" Website (The B.L.U.E. System)](https://www.youtube.com/watch?v=qcRKmm3B25c) . He proposes a free education system that teaches practical skills through how guides contributed by the community.

> The aim of the system is free universal education.

Therefore this MVP aims to explores how structured knowledge can exist without central ownership, while still remaining coherent, navigable, and meaningful.

It separates:
- data ownership
- knowledge interpretation
- system-wide structure

into independent layers that can evolve independently.
## Core Idea
Traditional learning systems assume knowledge must be:
- centrally stored
- centrally structured
- centrally ranked

B.L.U.E. challenges this assumption.

> Knowledge does not require a single authority to define its structure or progression.

Instead:
- data is owned at the edge (PDS)
- structure is derived (Indexers)
- interpretation is plural (multiple indexers can exist)

## Learning Unit Model - Articles
In B.L.U.E, an article is not a generic document it is a procedural learning guides

> An article is a structured “how-to guide” for understanding or doing something within a defined difficulty level.

Each article:
- teaches a specific concept or skill
- exists within a subject area
- belongs to a strict progression level
- is independently understandable within its constraints

#### Example forms of an article:
- “How to set up a Linux web server”
- “How HTTP requests work internally”
- “How to design a basic database schema”

They are not:
- essays
- open-ended discussions
- unstructured knowledge dumps

### Media expansion
While the MVP articles are text-based, the system is designed to expand into:
- images (diagrams, step illustrations)
- video walkthroughs

This is a structural evolution, not a conceptual change.

The definition remains:
> An article is a guided explanation of how to achieve understanding or perform a task.

Only the medium changes.

## Design Philosophy
### 1. User-owned data is fundamental
All content originates in a user-controlled PDS.

This ensures:
- ownership remains local
- portability is guaranteed
- no platform dependency exists

### 2. Structure is derived, not stored
The system does not assume a global curriculum exists.

Instead:
- structure is computed from distributed data
- hierarchy is reconstructed
- relationships are emergent, not predefined

> The system describes knowledge rather than defining it.

### 3. Interpretation is decentralised
Different indexers may produce different structural views of the same dataset.
This ensures:
- no single ranking authority
- multiple educational interpretations can coexist
- structural diversity is a feature, not a conflict

### 4. Meaning is not centrally controlled
No component of the system determines:
- importance
- ranking
- visibility
- canonical ordering

All of these are derived externally.

### 5. Knowledge is composable
Learning is not linear.
Instead it is:
- modular (articles)
- relational (methods)
- cross-linked (PDS aggregation)

This enables multiple learning paths through the same space.

## What B.L.U.E. is NOT
B.L.U.E. is explicitly not:
- a traditional Learning Management System (LMS)
- a centralised curriculum system
- a social feed platform
- a blockchain registry
- a single-source educational authority

It does not define truth.
It defines structure over distributed knowledge.

## Long-term Direction (non-MVP)
Future extensions may include:
- multiple independent indexers with different interpretations
- institutional and community-run nodes
- trust and reputation layers
- federated learning ecosystems
- real-time distributed update systems (relay layer)

However these are extensions, not requirements.

## Educational Philosophy
B.L.U.E. is built on a shift in focus:
> From content delivery to structured understanding

Traditional systems optimise for:
- completion
- certification
- linear progression

B.L.U.E. explores:
- conceptual structure
- relationships between ideas
- alternative explanations of the same knowledge
- emergent learning graphs

## Key Principles
- Data is owned at the edge (PDS)
- Articles are procedural learning guides
- Structure is computed externally (Indexers)
- Interpretation is non-exclusive
- No global authority defines knowledge
- Learning is a graph, not a pipeline


B.L.U.E. is not trying to replace existing educational systems.

It is testing a different assumption: What if structured learning could emerge without central control?
